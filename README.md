## Тестовое задание на позицию Golang-разработчика

Нужно создать простой бэкенд веб-сервиса на языке Go, который отдаёт файлы из папки `static` и возвращает ошибку 429 при превышении лимита запросов (10 в минуту) для подсети пользователя.

Требования к сервису:
1. Любой фреймворк, но предпочтительно стандартная библиотека Go.
2. Сервис должен работать на порту, который указывается как флаг при запуске.
3. Сервис должен принимать запросы на получение статического контента из папки `static`.
4. Сервис должен возвращать ошибку 429 (Too Many Requests) при превышении лимита запросов.
5. Сервис должен поддерживать извлечение IP-адреса из заголовка `X-Forwarded-For`.
6. Лимит должен быть 10 запросов в минуту.

Необязательные усложнения, которые будут плюсом:
- Юнит-тесты.
- Использование Redis для хранения лимитов.
- Поддержка нескольких подсетей с разными лимитами.
- Контейнеризация приложения с помощью Docker.
- Сброс лимитов через отдельный эндпоинт.

Прислать результат предпочтительно в виде ссылки на репозиторий на GitHub, но можно и просто архивом.
